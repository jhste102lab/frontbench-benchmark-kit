# FrontBench Prompt v1

Build a production-quality frontend for this scenario:

Scenario: "InvoicePulse", a lightweight accounts receivable dashboard for small B2B teams.

Requirements:
- Return one complete `index.html` file.
- Use only inline HTML, CSS, and JavaScript in that file.
- Do not load remote scripts, fonts, images, iframes, or trackers.
- Make the first screen an actual working product surface, not a marketing landing page.
- Include a dense but readable dashboard with receivables summary, overdue invoices, customer risk, cash forecast, and follow-up actions.
- Include at least three interactive controls such as filters, tabs, sorting, search, toggles, or buttons that update visible UI.
- Support desktop and 390px mobile widths.
- Keep contrast readable in light mode.
- Use realistic English UI copy.
- Avoid placeholder text such as lorem ipsum.
- Avoid explaining the implementation inside the UI.

Output contract:
- Return only the complete HTML document.
- Start with `<!doctype html>`.
- Do not wrap the HTML in Markdown fences.
